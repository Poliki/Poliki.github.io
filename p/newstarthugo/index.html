主要是重装后配的hugo都么有了，这次重新看以前的笔记和视频来配置一次

建博初衷

无限听说"程序员拥有自己的技术博客是一件非常重要的事情"

初识Hugo
官方主页：

https://gohugo.io

主题库：

https://themes.hugo.io

介绍

hugo 是基于 Go 语言的静态网站生成器。有两种方式发布生成的静态网站文件：

1.放到自己的服务器上提供服务：需要自己购买服务器 2.把网站托管到 GitHub Pages：需要将静态页面文件 push 到 GitHub 的博客项目的 gh-pages 分支并确保根目录下有 index.html 文件。

安装Hugo
1.官网下载

https://github.com/gohugoio/hugo/releases

找到Windows的版本进行安装，全程科学上网提速

2.配置环境变量 [ Win 10 ]

环境变量是经常配错的了，用户变量&系统变量搅浑是常态

这次亲测是在系统变量下，选择path -> 新建 -> 把刚解压的文件路径复制进去

如我的是：

D:\hugo_0.76.5
3.查看版本

CMD下输入hugo version就能查看到版本信息

生成本地站点
1.Git

后面的命令基于Git，没有安装的话去看 ”Git的安装与使用“

2.生成站点

在你的安装目录下，我的是 D:\hugo_0.76.5，右键 git bash here

$ hugo new site 站点名称

然后就会生成这个”站点名称“文件夹了

3.设置主题

去主题库的网站（上文有）中找到你喜欢的，下载然后解压到站点名称/themes 文件夹中，有的会让你直接git下载，不过推荐去Github链接那里下快一点，详细的可以看每个主题下的说明文档

然后点进站点文件再 git bash here 执行

$ hugo server -t 你的主题名称 --buildDrafts

然后就会以这个主题来启动本地项目了，往下拉就能看到Web Server is available at http://localhost:xxxx/

打开浏览器输入链接就能看到主题了

4.上传文章
$ hugo new post/yourblog.md

这个就是在post文件夹下新建了一个markdown文件

实际路径是 站点名称/content/post 中

然后编辑里面的md，重新开一次服务

$ hugo server -t 你的主题名称 --buildDrafts

一定要用new的方式建博客，直接复制一个md进去是没有用的，不过删是可以直接删

部署到远端
1.GitHub 新建一个仓库

！！！这个仓库的名字一定要是 你的github昵称.github.io

我的就是 Poliki.github.io

2.敲一堆命令
$ hugo --theme=你的主题名 --buildDrafts --baseUrl="https://你的昵称.github.io/"

然后就会给你生成一个public文件夹

3.把public文件夹传上去Github

没有用过Github建仓库的建议先去看“Github的使用”

在public目录下 git bash

$ git init   //初始仓库
$ git add .  //添加文件到缓存区
$ git commit -m "这里是描述信息"  //提交
如果这里你之前没有设置过git初始化会叫你填信息
$ git config --global user.email "you@example.com"
$ git config --global user.name "Your Name"
如果有就没事了
$ git remote add origin https://github.com/Polik/Poliki.github.io.git
//关联 按格式改地址
$ git push https://github.com/Poliki/Poliki.github.io master
//推送上去 master是Github以前的默认分支，现在叫main了，问题不大改一下就好

然后就可以访问一下

你的昵称.github.io.git检验成果了
